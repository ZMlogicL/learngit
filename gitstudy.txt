git status:
	用于查看当前仓库的状态。

git diff:
	查看文件修改部分。

git log:查看历史版本

git remote add origin git@github.com:ZMlogicL/lerangit.git：将本地仓库和远程仓库关联

如果远程库显示已存在，使用git remote rm origin可以删除

将本地库提交到远程库：git push -u origin master

从远程库克隆版本库到本地：git clone git@github.com ZMlogicL/learngit.git

git支持多种协议：http,ssh等ssh速度最快


2:git的分支管理。
	git创建分支命令：git checkout -b dev  -b参数表示创建并切换到分支
	等价于：git branch dev 
	              git checkout dev
git branch:查看当前分支

合并分支：需要先切换到主分支，然后执行git merge dev命令

git branch -d dev:删除分分支
创建并切换分支:git switch -c dev，直接切换不加参数c

git 更新远程分之到本地
　   １查看远程仓库
    	git remote -v
    ２从远程获取最新版本到本地
    	git fetch origin aaa
    ３比较远程分支和本地分支
     	git log -p aaa origin/aaa
    ４合并远程分支到本地
    	git merge origin/aaa

　　　git pull origin 分之名

当Git无法自动合并分支时，就必须首先解决冲突。解决冲突后，再提交，合并完成。

解决冲突就是把Git合并失败的文件手动编辑为我们希望的内容，再提交。

用git log --graph命令可以看到分支合并图
